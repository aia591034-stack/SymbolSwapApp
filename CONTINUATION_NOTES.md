# プロジェクト継続ノート: symbol-swap-app

## 1. プロジェクト概要
Symbol SDK v3 を使用した、アトミックスワップ形式のファイル売買マーケットプレイス。
- **A (運営)**: 手数料を支払うアグリゲートトランザクションの署名者。
- **B (ユーザーA)**: 出品者（または購入者）。
- **C (ユーザーB)**: 購入者（または出品者）。

## 2. 現在のステータス
- **修正完了**: 
    - SDK v3 のバイナリ化エラー (`offset is out of bounds`) 回避。
    - 署名検証エラー (`Ineligible_Cosignatory`) 解消。
    - ネットワーク禁止エラー (`V1_Prohibited`) 解消。
    - 接続先を `https://sym-test-01.opening-line.jp:3001` に統一。
    - `index.html` 内での公開鍵指定バグ（非公開鍵を渡していた）を修正。
    - アカウントA, B, C の残高確認完了（いずれも十分な残高あり）。
    - **Symbol SDK v3 の読み込み修正**: サーバー側は v3、フロントエンド（SSS連携用）は v2 を使用するハイブリッド構成に変更。
- **アグリゲートトランザクション V2 完全対応 (2026-04-22 深夜)**:
    - **SDK v3 の型定義修正**: `RangeError (not an integer)` や `PrivateKey is not a constructor` などのインポート/型エラーをすべて解消。
    - **シリアライズエラー修正**: `RangeError (offset is out of bounds)` を、`attachSignature` を使用したサイズ再計算により解消。
    - **堅牢な SSS 署名フロー**: `index.html` にて、SSS のバージョンや設定に依存しない 3段階のフォールバック署名ロジックを実装。
    - **バグ修正**: SSS 署名成功後の `checkStatus` 呼び出し時の未定義変数バグを修正。
    - **環境構築**: Vercel への環境変数設定（`OPERATOR_KEY`, `PINATA_API_KEY`等）を完了し、IPFS 永続保存を有効化。
- **現在のサーバーバージョン**: `1.1.2`
