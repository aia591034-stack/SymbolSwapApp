# プロジェクト継続ノート: symbol-swap-app

## 1. プロジェクト概要
Symbol SDK v3 を使用した、アトミックスワップ形式のファイル売買マーケットプレイス。
- **A (運営)**: 手数料を支払うアグリゲートトランザクションの署名者。
- **B (ユーザーA)**: 出品者（または購入者）。
- **C (ユーザーB)**: 購入者（または出品者）。

## 2. 現在のステータス
- **修正完了**: 
    - SDK v3 のバイナリ化エラー (`offset is out of bounds`) 回避。
    - 署名検証エラー (`Ineligible_Cosignatory`) 解消。
    - ネットワーク禁止エラー (`V1_Prohibited`) 解消。
    - 接続先を `https://sym-test-01.opening-line.jp:3001` に統一。
    - `index.html` 内での公開鍵指定バグ（非公開鍵を渡していた）を修正。
    - アカウントA, B, C の残高確認完了（いずれも十分な残高あり）。
    - **Symbol SDK v3 の読み込み修正**: サーバー側は v3、フロントエンド（SSS連携用）は v2 を使用するハイブリッド構成に変更。これにより SSS との完全な互換性を確保。

- **現在のステータス**:
    - サーバー起動中 (`http://localhost:3000`)。
    - SSS連携による購入フローが、最も安定した v2 形式で実行可能。

## 3. 次のステップ
1.  **ブラウザで動作確認**: `http://localhost:3000` を開き、SSSと連携して購入テストを行う。
    - 画面リロード（Ctrl+F5）を推奨。
    - 署名後に「アナウンス成功！」と表示されれば成功。
2.  **今後の課題**: SSS 拡張機能が SDK v3 に完全対応した段階で、フロントエンドも v3 に移行する。
